## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n log n) - 이 쿼리는 서브쿼리에서 Orders 테이블과 Products 테이블을 조인하고, 그룹화 후에 필터링을 수행합니다. 이 과정에서 데이터를 정렬하고 그룹화하는 시간이 가장 많이 소요되므로, 시간 복잡도는 O(n log n)입니다.

2. **공간 복잡도**: O(n) - 쿼리 결과를 저장하기 위한 공간이 필요합니다. 결과 집합의 크기는 입력 데이터의 크기에 비례하므로, 공간 복잡도는 O(n)입니다.

3. **풀이 접근법**: 이 쿼리는 서브쿼리에서 Orders 테이블과 Products 테이블을 조인하고, 그룹화하여 각 제품의 주문량을 계산합니다. 이후에 필터링을 수행하여 주문량이 100 이상인 제품만을 반환합니다. 사용된 알고리즘은 기본적으로 관계형 데이터베이스의 집계 함수와 조인 연산을 이용한 것입니다.

4. **잘된 점**: 코드는 간결하고, 읽기 쉽습니다. 서브쿼리를 사용하여 각 제품의 주문량을 계산하고, 메인 쿼리에서 필터링을 수행하여 결과를 반환하는 방식이 직관적입니다.

5. **개선 사항**: 쿼리에서 사용된 날짜 형식이 문자열로 지정되어 있습니다. Oracle SQL에서 날짜를 비교할 때는 DATE 타입을 사용하는 것이 좋습니다. 또한, BETWEEN 연산자는 날짜의 시간 부분을 고려하지 않으므로, '20200229 23:59:59'까지의 데이터를 포함하려면 날짜 범위를 조정해야 합니다. 예를 들어, `AND T2.order_date >= DATE '2020-02-01' AND T2.order_date < DATE '2020-03-01'`와 같이 지정할 수 있습니다.
