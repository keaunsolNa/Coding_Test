## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n^2) - 이 쿼리는 Prices 테이블과 UnitsSold 테이블을 조인하여 데이터를 가져옵니다. 조인 연산의 시간 복잡도는 일반적으로 O(n^2)입니다. 그러나 실제 시간 복잡도는 데이터베이스의 인덱스 설정, 데이터 분포 등에 따라 달라질 수 있습니다.

2. **공간 복잡도**: O(n) - 쿼리 결과는 product_id와 cnt를 포함하는 행으로 구성됩니다. 결과 집합의 크기는 입력 데이터의 크기에 비례하므로 공간 복잡도는 O(n)입니다.

3. **풀이 접근법**: 이 쿼리는 내부 조인(inner join)과 집계 함수(sum)를 사용하여 평균 판매 가격을 계산합니다. 조인은 두 테이블의 공통 열을 기준으로 행을 결합하고, 집계 함수는 결합된 행의 값을 집계하여 결과를 도출합니다.

4. **잘된 점**: 쿼리는 간결하고 읽기 쉽습니다. 필요한 열만 선택하고, 조인 조건과 집계 함수를 명확하게 지정하여 의도한 결과를 도출합니다.

5. **개선 사항**: 쿼리에서 사용된 조인 방식은 내부 조인입니다. 이는 두 테이블의 공통 열이 NULL이 아닌 경우에만 행을 결합합니다. 만약 NULL 값을 포함한 행도 결합해야 하는 경우, 왼쪽 조인(left join) 또는 전체 외부 조인(full outer join)을 사용해야 합니다. 또한, 쿼리에서 사용된 집계 함수는 sum입니다. 이는 평균 값을 계산하는 데 사용되므로, avg 함수를 사용하는 것이 더 적절할 수 있습니다. 예를 들어, `avg(T2.units * T1.price)`와 같이 평균 판매 가격을 계산할 수 있습니다.
