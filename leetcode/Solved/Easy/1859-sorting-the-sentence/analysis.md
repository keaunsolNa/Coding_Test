## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n log n)입니다. 여기서 n은 입력 문자열에 포함된 단어의 개수입니다. `TreeMap`을 사용하여 단어를 정렬하는 데 O(n log n) 시간이 소요됩니다. 또한 문자열을 분할하고 반복문에서 처리하는 데 O(n) 시간이 추가로 소요됩니다.

2. **공간 복잡도**: O(n)입니다. `TreeMap`에 단어를 저장하는 데 O(n) 공간이 소요됩니다. 여기서 n은 입력 문자열에 포함된 단어의 개수입니다.

3. **풀이 접근법**: 이 코드는 정렬 알고리즘을 사용하여 단어를 정렬합니다. `TreeMap`을 사용하여 단어를 키(인덱스)로 저장하고, 자동으로 정렬된 상태로 유지합니다. 이후에 키를 순서대로 조회하여 정렬된 단어를 얻습니다.

4. **잘된 점**: 코드는 읽기 쉽고, 변수명이 명확합니다. 또한 `TreeMap`을 사용하여 단어를 정렬하는 방법은 직관적이고 효율적입니다.

5. **개선 사항**: 코드는 이미 효율적이지만, 일부 최적화를 고려할 수 있습니다. 예를 들어, `StringBuilder`를 사용하여 문자열을 조합하는代わりに, `String`의 `+` 연산자를 사용하여 문자열을 연결할 수 있습니다. 또한, `TreeMap`의 키를 `Integer`로 사용하는 대신, `String`의 마지막 문자를 직접 추출하여 정렬할 수 있습니다. 하지만 이러한 최적화는 코드의 가독성을 저하할 수 있으므로, 필요할 때만 적용하는 것이 좋습니다.
