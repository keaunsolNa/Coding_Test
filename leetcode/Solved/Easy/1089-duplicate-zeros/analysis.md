## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n) - 이 코드는 입력 배열을 한 번 순회하고, 리스트를 한 번 순회하므로 시간 복잡도는 선형입니다. 여기서 n은 입력 배열의 크기입니다.

2. **공간 복잡도**: O(n) - 코드는 입력 배열의 크기와 동일한 크기의 리스트를 생성하므로 공간 복잡도도 선형입니다.

3. **풀이 접근법**: 이 코드는 두 가지 주요 단계로 구성됩니다. 첫째, 입력 배열을 순회하여 0을 발견할 때마다 리스트에 두 개의 0을 추가하고, 0이 아닌 요소는 단순히 리스트에 추가합니다. 둘째, 생성된 리스트를 원래 배열에 복사합니다. 이는 기본적인 리스트 조작과 배열 복사를 사용한 간단한 알고리즘입니다.

4. **잘된 점**: 코드는 읽기 쉽고, 변수명이 명확하며, 로직이 간단해서 이해하기 쉽습니다. 또한, 리스트를 사용하여 문제를 해결하려는 접근법은 직관적입니다.

5. **개선 사항**: 현재 코드는 추가적인 리스트를 생성하여 공간 복잡도를 높입니다. 더 최적화된 접근법은 두 개의 포인터를 사용하여 배열을 직접 조작하는 것입니다. 하나의 포인터는 읽기 위치를 가리키고, 다른 포인터는 쓰기 위치를 가리킵니다. 0을 만나면 쓰기 포인터를 사용하여 0을 복사하고, 쓰기 포인터를 이동시킵니다. 이렇게 하면 추가적인 공간을 사용하지 않아 공간 복잡도를 O(1)로 개선할 수 있습니다.
