## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n) - 이 쿼리는 Users 테이블의 모든 행을 스캔해야 하므로, 시간 복잡도는 테이블의 행 수에 비례합니다. REGEXP_LIKE 함수는 각 행에 대해 상수 시간을 소요하므로, 전체 시간 복잡도는 O(n)입니다.

2. **공간 복잡도**: O(n) - 쿼리 결과를 저장하기 위해 필요한 공간은 테이블의 행 수에 비례합니다. 따라서 공간 복잡도도 O(n)입니다.

3. **풀이 접근법**: 이 쿼리는 정규 표현식을 사용하여 이메일 주소를 필터링합니다. REGEXP_LIKE 함수는 주어진 패턴과 일치하는 문자열을 찾는 데 사용됩니다. 이 접근법은 간단하고 직관적이며, Oracle SQL에서 지원하는 기능을 잘 활용하고 있습니다.

4. **잘된 점**: 코드는 간결하고 읽기 쉽습니다. REGEXP_LIKE 함수를 사용하여 이메일 주소를 필터링하는 것은 적절한 방법입니다. 또한, ORDER BY 절을 사용하여 결과를 정렬하는 것은 좋은 관행입니다.

5. **개선 사항**: 이메일 주소의 유효성을 검사하는 패턴은 간단한 경우에만 적용됩니다. 실제로 사용되는 이메일 주소의 유효성 검사에는 더 복잡한 규칙이 적용될 수 있습니다. 예를 들어, 이메일 주소에는 하이픈(-)이나 마침표(.)가 포함될 수 있습니다. 이러한 경우를 처리하기 위해 더 복잡한 정규 표현식을 사용할 수 있습니다. 또한, 이 쿼리는 대소문자를 구분하여 이메일 주소를 필터링합니다. 대소문자 구분 없이 필터링하려면 REGEXP_LIKE 함수의 세 번째 인자로 'i'를 지정할 수 있습니다.
