## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n + m)입니다. 여기서 n은 보석 문자열(jewels)의 길이이고, m은 돌 문자열(stones)의 길이입니다. 두 문자열을 각각 한 번씩 순회하기 때문입니다.

2. **공간 복잡도**: O(1)입니다. 보석 여부를 확인하기 위해 128개의 크기를 가진 배열을 사용했는데, 이는 입력의 크기와 상관없이 항상 같은 크기이기 때문입니다.

3. **풀이 접근법**: 해시 테이블(배열)을 사용하여 보석 여부를 빠르게 확인했습니다. 각 보석 문자를 배열의 인덱스로 사용하여 해당 문자가 보석인지 아닌지 바로 확인할 수 있습니다.

4. **잘된 점**: 코드가 간결하고 이해하기 쉽습니다. 또한, 배열을 사용하여 보석 여부를 확인하는 방법이 효율적입니다. 변수명도 의미가 명확하여 코드의 가독성이 좋습니다.

5. **개선 사항**: 코드는 이미 khá 효율적이지만, 자바의 `HashSet`을 사용하여 보석 문자열을 저장하고 돌 문자열을 순회하며 확인하는 방법도 있습니다. 하지만, 현재의 배열 기반 접근법이 더 빠를 수 있습니다. 또한, 주석을 추가하여 코드의 목적과 로직을 설명하면 코드의 이해도를 높일 수 있습니다.
