3. **풀이 접근법**: 이 풀이는 간단한 반복문을 사용하여 각 보석 문자가 돌 문자열에 몇 번 등장하는지 확인합니다. 이는 브루트 포스 알고리즘의 예입니다.

5. **개선 사항**: 현재 풀이는 각 보석 문자마다 돌 문자열을 전체扫描하기 때문에 시간 복잡도가 높습니다. 이를 개선하기 위해 해시 테이블을 사용하여 돌 문자열에 등장하는 각 문자의 빈도를 미리 계산한 후, 보석 문자열의 각 문자에 대해 해당 빈도를 바로 조회할 수 있습니다. 예를 들어, `HashSet`을 사용하여 보석 문자열의 각 문자를 저장하고, 돌 문자열을扫描하여 각 문자가 보석 문자열에 포함되는지 확인할 수 있습니다. 이렇게 하면 시간 복잡도를 O(n + m)으로 개선할 수 있습니다.
