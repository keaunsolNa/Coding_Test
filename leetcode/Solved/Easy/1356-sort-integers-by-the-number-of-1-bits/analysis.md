## Code Review

코드 리뷰입니다.

1. **시간 복잡도**: O(n log n)입니다. 이유는 두 번의 반복문이 있지만, 각 반복문은 O(n)의 시간 복잡도를 가지며, 중간에 Arrays.sort() 함수가 호출되어 O(n log n)의 시간 복잡도를 가지기 때문입니다. Arrays.sort() 함수는 자바에서 내장된 정렬 함수로, 일반적으로 퀵소트나 머지소트와 같은 효율적인 알고리즘을 사용합니다.

2. **공간 복잡도**: O(1)입니다. 이유는 입력 배열을 제외하고 추가적인 공간을 사용하지 않기 때문입니다. 입력 배열은 문제의 입력으로 주어지므로, 공간 복잡도 계산에 포함되지 않습니다.

3. **풀이 접근법**: 이 코드는 비트 카운팅과 정렬을 사용하여 문제를 해결합니다. 먼저 각 숫자의 비트를 카운팅하여 숫자에 비트 카운트를 추가적으로 저장하고, 이후 정렬을 통해 비트 카운트가 적은 순으로 정렬합니다. 이후에 비트 카운트를 제거하여 원래의 숫자만 남깁니다.

4. **잘된 점**: 코드는 간결하고, 문제의 요구 사항을 명확하게 이해하여 구현했습니다. 또한, 비트 카운팅과 정렬을 사용하여 문제를 효율적으로 해결했습니다.

5. **개선 사항**: 코드는 이미 효율적이지만, 더 나은 접근법은 Comparator를 사용하여 정렬하는 것입니다. 이는 코드를 더 명확하게 만들어주고, 비트 카운트를 따로 저장할 필요가 없기 때문입니다. 예를 들어, Arrays.sort()에 Comparator를 전달하여, 비트 카운트와 숫자를 동시에 고려하여 정렬할 수 있습니다. 또한, 매직 넘버인 100000을 상수로 정의하여 코드의 가독성을 높일 수 있습니다.
